# A - King's Tour 要件仕様書

## 1. 概要

AtCoder 王国において、国王が `N × N` の全区画をちょうど一度ずつ訪問するツアーを計画する。

各区画には住民数が設定されており、後の日に訪問するほど高い好感度を得られる。  
そのため、住民数が多い区画をなるべく後半に訪問することが望ましい。

ただし、連続する訪問区画は「縦・横・斜め」に隣接している必要がある。

本問題では、制約を満たしつつ、総好感度を最大化する訪問順序を求める。

---

# 2. 用語定義

| 用語 | 説明 |
|---|---|
| 区画 | グリッド上の 1 マス |
| `(i, j)` | 上から `i`、左から `j` の位置 |
| `A[i][j]` | 区画 `(i,j)` の住民数 |
| ツアー | 全区画を一度ずつ巡回する訪問順 |
| `k 日目` | `0 ～ N^2-1` の訪問順序 |

---

# 3. 入力仕様

## 3.1 入力形式

```text
N
A0,0 A0,1 ... A0,N-1
A1,0 A1,1 ... A1,N-1
...
AN-1,0 ...     AN-1,N-1
```

---

## 3.2 パラメータ

### `N`

グリッドの一辺の長さ。

```text
グリッドサイズ = N × N
```

---

### `A[i][j]`

区画 `(i,j)` に住む国民数。

- 各区画で値は異なる
- 非負整数

---

# 4. 出力仕様

## 4.1 出力内容

各日ごとに訪問する区画座標を出力する。

```text
i0 j0
i1 j1
...
i(N^2-1) j(N^2-1)
```

---

## 4.2 出力条件

### 全区画をちょうど一度ずつ訪問

```text
訪問回数 = N^2
```

かつ

```text
全ての区画が一意
```

---

### 隣接条件

連続する 2 日の区画について：

```text
max(
    |ik - ik+1|,
    |jk - jk+1|
) = 1
```

を満たす必要がある。

---

## 4.3 許可される移動

以下の 8 方向への移動が可能：

```text
↑ ↓ ← →
↖ ↗ ↙ ↘
```

---

# 5. 評価関数

## 5.1 総好感度

`k 日目` に訪問した区画 `(ik, jk)` に対して：

```text
獲得好感度 = k × A[ik][jk]
```

総好感度 `V` は：

```text
V = Σ (k × A[ik][jk])
    k=0..N^2-1
```

---

# 6. スコア計算

各テストケースのスコア：

```text
score = round(V / N^2)
```

---

## 6.1 提出スコア

```text
全100ケースの score の総和
```

---

# 7. 最適化方針

## 7.1 基本戦略

住民数が多い区画ほど後半に訪問するとスコアが高くなる。

つまり：

```text
小さい A → 前半
大きい A → 後半
```

となる順序が望ましい。

---

## 7.2 ただし存在する制約

単純なソート順では移動制約を満たさない。

そのため：

- Hamilton Path 的制約
- グリッド隣接制約
- スコア最大化

を同時に満たす必要がある。

---

# 8. 問題の本質

本問題は以下を組み合わせた最適化問題である。

## 8.1 経路制約

```text
全頂点を一度ずつ訪問
```

かつ

```text
隣接マス移動のみ
```

---

## 8.2 重み付き順序最適化

```text
後ろに置くほど得点が大きい
```

ため、

```text
高人口マスを後半へ配置
```

する必要がある。

---

# 10. 制約上の注意

## 10.1 不正解条件

以下の場合は失格：

- 同一区画を複数回訪問
- 未訪問区画が存在
- 非隣接移動
- 実行時間超過

---

# 11. 実装上の推奨事項

## 11.1 データ構造

```cpp
vector<pair<int,int>> path;
```

---

## 11.2 高速化ポイント

- スコア差分計算
- 近傍更新
- 経路妥当性の局所判定

---

# 12. 目的

以下を最大化すること：

```text
Σ k × A[path[k]]
```

ただし：

```text
path は隣接移動による Hamilton Path
```

である必要がある。
